================================================================================
RESULTS INTERPRETATION
================================================================================

PAIRWISE IDENTITY:
  Value between 0 and 1 (or 0% to 100%)
  0.95 = 95% identical
  0.50 = 50% identical
  0.05 = Only 5% identical (very distant)

AVERAGE PAIRWISE IDENTITY:
  > 0.70 = Close homologs (same family)
  0.30-0.70 = Divergent but related
  < 0.30 = Very distant (possibly unrelated)

FRACTION CONSERVED:
  > 0.50 = Highly conserved (stable structure)
  0.10-0.50 = Mixed conservation
  < 0.10 = Rapidly evolving (variable)

================================================================================
ALGORITHMS AT A GLANCE
================================================================================

1. Needleman-Wunsch Algorithm
   - Pairwise global sequence alignment
   - Uses dynamic programming
   - BLOSUM62 scoring matrix
   - Affine gap penalties

2. UPGMA Clustering
   - Builds guide tree for alignment order
   - Hierarchical clustering approach
   - Average linkage distance metric

3. Progressive MSA
   - Align sequences following guide tree
   - Combine pairwise alignments
   - Optional refinement step

4. Analysis
   - Pairwise identity calculation
   - Conservation identification
   - Formatted output generation

================================================================================
TYPICAL WORKFLOW
================================================================================

Step 1: Prepare your sequences
  - Format as FASTA (standard biology format)
  - Each sequence should be from the same family
  - Headers should describe sequences (>protein_name)

Step 2: Run the alignment
  python msa_task.py --input myseqs.fasta --output myseqs_aligned.fasta

Step 3: Check the metrics
  - Read console output for statistics
  - Identity scores tell you about sequence similarity
  - Conservation tells you about functional regions

Step 4: Use the aligned output
  - View in Jalview or SeaView (alignment viewers)
  - Create phylogenetic tree (FastTree, IQ-TREE)
  - Predict structure (AlphaFold, SWISS-MODEL)
  - Find domains (InterProScan, Pfam)

Step 5: Interpret results
  - High identity = Closely related sequences
  - High conservation = Functionally important residues
  - Low identity = Distant sequences, may not be homologous

================================================================================
COMMON QUESTIONS
================================================================================

Q: I got very low identity scores (all near 0)
A: Your sequences may not be homologous (from same family).
   → Double-check sequence source
   → Verify they're from the same protein family
   → See QUICKSTART.md for more details

Q: How do I make it faster?
A: For large datasets:
   → Use fewer sequences
   → Skip alignment refinement (comment out line in code)
   → Use external tools (Clustal, MAFFT) for 100+ sequences

Q: Can I use DNA/RNA?
A: Script is optimized for proteins.
   For DNA/RNA, you'd need to remove BLOSUM62 dependency.

Q: How do I customize gap penalties?
A: Edit msa_task.py, find line with needleman_wunsch()
   Change gap_open and gap_extend parameters
   See TECHNICAL.md for details

Q: Can I integrate this into my pipeline?
A: Yes! The script is self-contained and designed for integration.
   See TECHNICAL.md for examples.

================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: "No sequences found"
FIX: Check FASTA format
    - Each sequence needs a header line starting with >
    - Sequence data on following lines
    - Example: 
      >seq_name
      MKVLIVFASF

PROBLEM: Script hangs or runs very slowly
FIX: Check dataset size
    - This script is O(n³) for tree building
    - For 100+ sequences, use external tools instead
    - Reduce number of sequences if possible

PROBLEM: ImportError: No module named 'numpy'
FIX: Install numpy
    pip install numpy

PROBLEM: "python: command not found" or version is 2.x
FIX: Install/update Python 3.11+
    https://www.python.org/downloads/

PROBLEM: "Permission denied" when running script
FIX: Make script executable
    chmod +x msa_task.py

PROBLEM: Output file not created
FIX: Check write permissions
    - Directory must be writable (chmod 755)
    - Specify full path if needed
    - Check disk space

More troubleshooting in QUICKSTART.md
